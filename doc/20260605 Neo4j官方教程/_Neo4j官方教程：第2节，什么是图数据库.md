## 一、图数据库核心定义

**图数据库是以「属性图模型」原生存储数据的数据库，数据由节点 \(Node\)、关系 \(Relationship\)、属性 \(Property\) 组成，和我们在草稿纸上手绘实体关系图结构完全一致**。

- 关系库：数据固化在数据表、行、字段，靠外键、JOIN 关联数据；

- 图数据库：实体是节点，实体之间的关联直接存为关系，**不需要额外关联表、不需要 JOIN 运算**。

> Cypher 示例可视化：`MATCH(:Person{name:"Dan"})-[:LOVES]->(:Person{name:"Ann"})`
> Dan（人员节点）【LOVES 相爱】→ Ann（人员节点），节点带标签 Person、属性 name。
> 
> 

## 二、四大基础组件详解（重中之重，对标 MySQL）

### 1\. 节点（Node）

代表现实世界中的**实体对象**：企业、用户、课程、商品、设备、人员都可以作为节点。

- 特点：

    1. 可绑定 1 个或多个`标签(Label)`做分类；

    2. 可自定义任意个键值对`属性`，没有固定字段约束（无 Schema）；

    3. 支持创建索引、唯一性约束，优化查询速度。

> 示例：`(e:Enterprise{entId:"001",name:"XX科技",credit:1000})`
> e：节点别名；Enterprise：标签；entId/name/credit：属性
> 
> 

### 2\. 标签（Label）

节点的分类标识，用来批量归类同类型节点，等价于 MySQL 的**数据表**。

- 一个节点可挂载多个标签：比如一个实体既是`Person`又是`Customer`；

- 查询时通过标签快速过滤：`MATCH(e:Enterprise)`只查企业类节点。

### 3\. 属性（Property）

节点 / 关系上的键值对数据，等价于 MySQL**字段 \+ 字段值**，灵活扩展，随时新增删除属性，不用改表结构。

> 企业节点新增`registerMoney`注册资金属性：直接 SET 赋值，无需 DDL 修改表。
> 
> 

### 4\. 关系（Relationship）

用来描述**两个节点之间的关联逻辑**，是图数据库最核心优势，必须满足 3 个硬性规则：

1. **必有方向**：起始节点→结束节点（A 投资 B ≠ B 投资 A）；

2. **必有唯一类型**：如 INVEST 投资、FRIEND 好友、SUPPLY 供货；

3. **可挂载属性**：关系自身也能存数据（例：INVEST \{money:5000 万，time:"2025\-01"\}）；

- 同一对节点可创建多条不同类型关系，不会损耗查询性能。

> 标准格式：`(A)-[r:INVEST{money:5000}]->(B)`
> 
> 

## 三、图数据库工作原理

1. 数据落地：实体→节点、分类→标签、详情→属性、关联→带方向关系；

2. 查询逻辑：顺着**原生存储的关系链路遍历数据**，没有多表 JOIN 开销；

3. 性能优势：不管链路是 3 层、10 层、20 层关联，遍历效率变化极小，关系越多对比 MySQL 优势越明显。

## 四、图数据库 VS 关系型数据库（MySQL）核心对比

|对比项|MySQL（关系数据库）|Neo4j（图数据库）|
|---|---|---|
|数据结构|数据表、行、字段，依靠外键 / 中间表关联|节点 \+ 有向关系 \+ 属性，关联原生存储|
|多深度关联查询|多层 JOIN，层级越多性能断崖下跌|顺着关系遍历，深度不影响性能|
|字段约束|建表固定字段，新增字段需要 ALTER 改表|无固定 Schema，随时新增属性|
|关联新增成本|多对多必须新建中间关联表|直接新增一条关系即可|
|适用场景|单表增删改查、简单统计|股权穿透、供应链、社交关系、知识图谱|

> 举例：查询 A 企业→母公司→母公司的投资方，3 级股权穿透：
> MySQL 需要 3 次 JOIN；Neo4j 直接顺着 INVEST 关系遍历，写法简单、性能稳定。
>

## 五、为什么项目要用图数据库？

1. **规避昂贵 JOIN 运算**：关系库多层关联查询计算成本极高，图数据库无 JOIN，原生存储关联；

2. **灵活的数据模型**：业务迭代新增字段、新增关联不用修改库结构；

3. **擅长深度链路查询**：找间接关联、隐藏关系是图数据库独有优势；

4. **海量节点横向扩展**：支持数十亿节点存储，适合大规模图谱业务。

**业务落地方向：凡是重点关注「实体之间关系」的项目，优先选用 Neo4j；只关注单表数据统计用 MySQL。**

## 七、随堂实操练习（基于 Cypher，浏览器 7474 直接运行）

1. 创建 2 个企业节点：`Enterprise{name:"甲公司",credit:1000}`、`Enterprise{name:"乙公司",credit:800}`；

2. 创建关系：甲公司【INVEST 投资 \{money:3000\}】→乙公司；

3. 编写 Cypher 查询：查询所有被甲公司投资的企业。

```cypher
//参考答案
CREATE (a:Enterprise{name:"甲公司",credit:1000})-[:INVEST{money:3000}]->(b:Enterprise{name:"乙公司",credit:800})
MATCH (a:Enterprise{name:"甲公司"})-[:INVEST]->(b) RETURN b.name,b.credit
```
